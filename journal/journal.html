Although most people dread pop-up windows, Modals and Overlays are essential in web design. A modal window is an element that sits on top of an application's main window, requiring users to interact with it before they return to the parent application. It needs to grab the user's attention, provide information, and allow users to input their own information.
A modal window includes. title, a CTA button, and an option to close the browser, all on top of a shadow. The sizing and location should be in the center and big enough to be legible, as well as provide context that they have not left the site; the location should be in the upper half of the screen because in mobile view, the modal may be lost if placed lower.
Modal Windows can help get the user's attention; however, it breaks the user's flow and make it impossible to interact with the parent page by hiding content behind the modal. Forms appear in every day websites, allowing brands or anyone behind the screens to survey the user to provide services. However, forms can exhaust many people since they need to provide a lot of information, therefore, providing a better UI such as indicating multiple steps, progress checker, or simplifying a column can ease the experience. In Salim Arsi’s blog, “Best practices for form design: 20 tips to boost form usability,” she provides many useful tips to increase accessibility and engagement such as providing a placeholder text below the textfield to hint users where to instruct users on where to type their answer.
Small psychological tactics like asking the easy questions first, or using field lengths as an affordance, can help make a pleasurable experience. Accessibility such as contrast, text size control, and color can help with anyone with disabilities. Ultimately, filling out an online form is not anyone’s pastime, however the goal of a product designer is to design a straightforward, practical form that is easy to fill in, allowing increase in usability.
Websites like AirBnB provide a good user interface when reserving a place to stay–what most people do not like doing. It is straight forward and indicates steps: Choose when to Pay, Adding a payment Method, and Reviewing the Reservation. There are clear CTA buttons like to Reserve or Pay, and make sure that the fieldboxes need to be required to allow users to write mandatory data for the website.
