Although the website appears visually simplistic and does not break creative barriers, my favorite website I like to visit is the Train Jazz website. Visually, the website holds its essentials: a map of the New York subways and colorful dots indicating the live location of the trains. However, its visuals interact with the listeners by combining both visual and aural elements to create a pleasant experience. The visuals appear minimal, so its interactive musical experience shines through. It uses a live map of subways in New York to create jazz, imitating the hustling, bustling, lively nature of New York and its people.
The designer states that a note is placed precisely where the train happens to be along its route: rush hours fill the band with held tones, and at 3 AM, the silence grows longer, creating a unique melodious symphony. When changing from map to bars, it transitions to a musical bar, where you can see where each train's line interacts with each other, forming notes. Furthermore, each line correlates with its instrument: The 1/2/3 uses an upright bass because these are the original subway lines, thus getting a foundational instrument for a jazz combo, while the W line plays a warm guitar because it has been discontinued and brought back several times, creating an on-again, off-again quality to the melody.
Ultimately, the website uses a minimal yet thoughtful design to combine both visual and aural elements, emphasizing the goal of the website: to show the beautiful interaction of New York’s subways and music.

Although most people dread pop-up windows, Modals and Overlays are essential in web design. A modal window is an element that sits on top of an application's main window, requiring users to interact with it before they return to the parent application. It needs to grab the user's attention, provide information, and allow users to input their own information.
A modal window includes. title, a CTA button, and an option to close the browser, all on top of a shadow. Modal Windows can help get the user's attention; however, it breaks the user's flow and make it impossible to interact with the parent page by hiding content behind the modal. Forms appear in every day websites, allowing brands or anyone behind the screens to survey the user to provide services. However, forms can exhaust many people since they need to provide a lot of information, therefore, providing a better UI such as indicating multiple steps, progress checker, or simplifying a column can ease the experience. In Salim Arsi’s blog, “Best practices for form design: 20 tips to boost form usability,” she provides many useful tips to increase accessibility and engagement such as providing a placeholder text below the textfield to hint users where to instruct users on where to type their answer.
Small psychological tactics like asking the easy questions first, or using field lengths as an affordance, can help make a pleasurable experience. Accessibility such as contrast, text size control, and color can help with anyone with disabilities. Ultimately, filling out an online form is not anyone’s pastime, however the goal of a product designer is to design a straightforward, practical form that is easy to fill in, allowing increase in usability.
Websites like AirBnB provide a good user interface when reserving a place to stay–what most people do not like doing. It is straight forward and indicates steps: Choose when to Pay, Adding a payment Method, and Reviewing the Reservation. There are clear CTA buttons like to Reserve or Pay, and make sure that the fieldboxes need to be required to allow users to write mandatory data for the website.
